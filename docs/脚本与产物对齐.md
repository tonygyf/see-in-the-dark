# 脚本与产物对齐

## 1) 数据准备脚本

| 脚本 | 输入 | 主要输出目录 | 关键产物 |
|---|---|---|---|
| `scripts/prepare_ctw1500_test.ps1` | CTW1500 官方测试集压缩包链接 | `data/raw/ctw1500/` | `imgs/test/`、`annotations/test/`、`downloads/*.zip` |
| `scripts/prepare_sid_sony_subset.ps1` | SID Sony2025 下载链接 | `data/raw/sid_sony_text/` | 默认 `-Mode selective`：只下载前1500图到 `subset_1500/images/`、`subset_1500/first1500.txt`、`subset_1500/manifest_first1500.csv`；`-Mode full` 才会生成 `downloads/Sony2025.zip` 与 `extracted/` |

说明：数据脚本产物统一在 `data/raw/...`，不写入 `runs/experiments`。

## 2) 训练启动脚本（产物必须对齐 runs/experiments）

| 脚本 | 配置 | 产物目录（对齐） |
|---|---|---|
| `scripts/start_laptop_train.ps1` | `configs/laptop_4060_quant_friendly.yaml` | 默认 `runs/experiments/default`（若配置未指定 experiment） |
| `scripts/start_laptop_train_realdata_smoke.ps1` | `configs/laptop_4060_realdata_ctw1500_smoke.yaml` | 默认 `runs/experiments/default`（建议后续在 config 增加 experiment.name/out_dir） |
| `scripts/start_laptop_train_realdata_1000step.ps1` | `configs/laptop_4060_realdata_ctw1500_1000step.yaml` | 默认 `runs/experiments/default`（建议后续在 config 增加 experiment.name/out_dir） |
| `scripts/start_laptop_train_realdata_high_intensity.ps1` | `configs/laptop_4060_realdata_ctw1500_high_intensity.yaml` | 默认 `runs/experiments/default`（建议后续在 config 增加 experiment.name/out_dir） |
| `scripts/start_laptop_train_sid_sony_subset1500_smoke.ps1` | `configs/laptop_4060_sid_sony_subset1500_smoke.yaml` | `runs/experiments/sid_sony_subset1500_smoke_4060` |

训练脚本统一产物：
- `starter_last.pt`
- `train_loss.csv`
- `train_loss_curve.png`
- `prediction_sample.png`

以上文件均写入对应 `runs/experiments/<experiment_name>/`。

## 3) 汇总脚本

| 脚本 | 输入 | 输出 |
|---|---|---|
| `scripts/summarize_scm_grid.py` | `runs/experiments/scm_grid_4060/*` 子实验目录 | `runs/experiments/scm_grid_4060/summary.csv`、`summary.md` |

## 4) 对齐约定（执行时遵循）

1. 数据准备只写 `data/raw/`，不污染 `runs/experiments/`。  
2. 每个训练配置应显式设置 `experiment.name` 和 `experiment.out_dir`，避免都写进 `default`。  
3. 每次新实验前，先把上一轮关键结果归档到 `runs/history/`。  
4. 训练结束后更新 `runs/experiments/summary_all.*`，保持总表可追踪。
